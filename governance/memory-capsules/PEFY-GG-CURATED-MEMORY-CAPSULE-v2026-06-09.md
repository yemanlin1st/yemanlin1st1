## 1. Capsule Purpose

This file preserves a curated, GitHub-safe operational memory for the PEFY-GG ecosystem. It is not a raw memory dump. It contains only reusable rules, execution standards, project families, branding principles, quality gates, architecture preferences, and protection logic that can safely guide future work.

The full confidential memory, private identity mapping, internal evidence register, personal identifiers, legal proofs, and sensitive client information must remain outside public repositories.

All public-facing assets must use protected coded identity instead of personal exposure.

Each relevant app, design, production, document, repository, or deployable package should have a project-specific QR Code.

Public QR data may contain:

- Project name
- Asset ID
- Version
- Owner Code
- IP Controller
- Public brand identity
- Public contact channel
- License/usage status
- Verification reference
- Hash/fingerprint
- Confidentiality level

Public QR data must not contain:

- Personal owner identity
- Private contacts
- Legal IDs
- Contract references
- Repository secrets
- Confidential registry data
- Sensitive client data
